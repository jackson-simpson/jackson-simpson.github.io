Differences between Relative, Absolute and Fixed Positioning

Our work in sprint 2 has been focused on learning and understanding how to use HTML and CSS. I found that HTML came easily to me, but when it came time to add CSS code, and get it to interact with the content on my webpage I hit a block. Understanding concepts like colour were fine, but I really had a lot of trouble with website responsiveness and element positioning. So for this part of my technical blog, I wanted to go over the research I've done into positioning, and how I now understand it a lot better.

The first thing to note is, there are more than the three types of positioning that I am going over in this post, which can also be useful in making a website feel more accessible. But to start we need to talk about the defult property.

Static Positioning

Static positioning is the basic, defult property that all elements have. Without any additional position properties, the elemental will act statically. This means it will generally be moving down the working axis of the page, typically vertically. This doesn't give us a lot to work with, so we add other positioning properties to class and ID elements to move them around the page.
